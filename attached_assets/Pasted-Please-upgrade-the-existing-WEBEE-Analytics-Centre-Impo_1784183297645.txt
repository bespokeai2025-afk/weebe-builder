Please upgrade the existing WEBEE Analytics Centre.

Important context:

The Analytics Centre already pulls some analytics from Retell AI. Do not remove or replace the existing Retell analytics integration. Instead, audit it, improve it, normalise the data, and combine it with WEBEE’s own CRM, leads, campaigns, workflows, follow-ups, bookings, costs, and reports.

The goal is to turn Analytics Centre into a deep business intelligence hub for each workspace.

Everything must be workspace-scoped, package-gated, role-permission checked, and safe.

Do not expose one workspace’s analytics to another workspace.

Do not break WBAH.

Keep it.

* Retell data is pulled using the current workspace’s Retell API key

# 18. Automated report sending

Reports should send according to workspace notification settings.

Email provider priority:

1. Workspace custom email provider if active and package allows.
2. Reseller parent custom provider if child inherits parent email mode and package allows.
3. WEBEE admin Resend account as default fallback.

Default should be WEBEE admin Resend.

Use:

* `WEBEE_RESEND_API_KEY`
* `WEBEE_DEFAULT_FROM_EMAIL`
* `WEBEE_DEFAULT_FROM_NAME`

Never expose the WEBEE Resend key to users.

If sending fails:

* mark as failed
* log error
* show in-app warning
* do not break campaign execution

# 24. Safety

Analytics must be read-only unless the user explicitly creates a draft action from an insight.

Analytics should never directly:

* trigger calls
* send messages
* alter campaigns
* alter agents
* alter workflows
* change CRM records
* change package settings
* change user access

Any action from analytics must go through SystemMind as a draft and require approval.
