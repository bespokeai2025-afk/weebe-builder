BUILD SYSTEMMIND WORKFLOW GENERATOR FOR BUILDER DRAFTS

IMPORTANT

Do not allow SystemMind to modify live deployed agents automatically.

Do not allow SystemMind to deploy agents without user approval.

Do not break existing Builder functionality.

Extend SystemMind and Builder safely.

OBJECTIVE

Allow users to describe a new conversational workflow in natural language and have SystemMind generate a Builder-compatible draft workflow.

Example user request:

"Create a rebooking workflow. If the caller wants a callback, collect preferred date and time, check calendar availability, confirm the slot, update CRM, and schedule the callback."

====================================================

REBOOKING WORKFLOW TEMPLATE

SystemMind should support rebooking workflows.

Required flow:

1. Detect callback or rebooking intent.
2. Ask for preferred date.
3. Ask for preferred time.
4. Extract date/time variables.
5. Confirm date/time back to caller.
6. Check calendar availability if connected.
7. If available, create booking or callback task.
8. If unavailable, offer alternatives.
9. Update lead/contact status.
10. Add CRM note.
11. Trigger follow-up message.
12. End call politely or return to main flow.

Required variables:

customer_name
phone_number
preferred_date
preferred_time
timezone
callback_reason
booking_status
calendar_event_id
crm_record_id

Required tools:

check_availability
create_booking_or_callback
update_lead_status
create_crm_note
send_confirmation_message

====================================================

Suggested Fix:

Risk:
Medium

Approval Required:
Yes

====================================================

APPROVAL RULES

SystemMind may create draft workflows.

SystemMind may not:

Deploy agents
Modify live agents
Change phone routing
Start outbound campaigns
Change production webhooks

without HiveMind approval.

HiveMind controls approval.

No live agent is modified automatically.

Generate implementation plan first.

Then implement safely.
