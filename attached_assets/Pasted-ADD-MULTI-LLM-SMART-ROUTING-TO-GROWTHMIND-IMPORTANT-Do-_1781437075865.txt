ADD MULTI-LLM SMART ROUTING TO GROWTHMIND

IMPORTANT

Do not modify Retell.
Do not modify HyperStream voice.
Do not modify HiveMind voice.
Do not replace OpenAI globally.
Do not break existing GrowthMind Content Studio.
Extend GrowthMind only.

OBJECTIVE

Add multi-LLM support to GrowthMind so different marketing tasks can use the best model automatically.

Supported providers:

1. OpenAI
2. Gemini
3. Claude

Supported models:

OpenAI:
- GPT-4.1
- GPT-4.1 Mini

Gemini:
- Gemini 2.5 Flash
- Gemini 2.5 Pro

Claude:
- Claude Sonnet 4

ENVIRONMENT VARIABLES

Support server-side only:

OPENAI_API_KEY
GEMINI_API_KEY
ANTHROPIC_API_KEY

Never expose keys to frontend.

CREATE GROWTHMIND MODEL ROUTER

Create:

src/lib/growthmind/model-router.server.ts

Responsibilities:

- Select provider
- Select model
- Route generation requests
- Support fallback model
- Track usage
- Track cost
- Return structured outputs

CREATE PROVIDER SERVICES

Create:

src/lib/growthmind/providers/openai-growth.server.ts
src/lib/growthmind/providers/gemini-growth.server.ts
src/lib/growthmind/providers/claude-growth.server.ts

Each provider must expose:

generateText()
generateJson()
generateContentAsset()
generateStrategy()
generateReport()

FALLBACK LOGIC

If selected model fails:

1. Retry once.
2. If still failing, use fallback model.
3. Log failure.
4. Show friendly message.

Default fallback:

Gemini 2.5 Pro → GPT-4.1
Claude Sonnet 4 → GPT-4.1
Gemini 2.5 Flash → GPT-4.1 Mini
GPT-4.1 → Gemini 2.5 Pro
