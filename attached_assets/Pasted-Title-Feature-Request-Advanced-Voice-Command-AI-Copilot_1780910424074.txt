Title: Feature Request - Advanced Voice Command AI Copilot Engine

Please implement a complete, production-grade "Voice Command AI Copilot" feature within our node builder workspace. This feature allows users to speak instructions into a microphone, which are parsed by an intelligent backend pipeline and translated into explicit frontend state function calls to dynamically assemble, label, modify, position, and delete items on the canvas.

---

### 1. UI/UX Interface Elements
* **The Microphone Trigger:** Add a sleek, compact microphone icon button directly into the workspace layout. It must match our dark theme and use our signature accent colors for active/hover states.
* **Recording Feedback Loop:** Clicking the button switches it to a "Listening..." state, accompanied by a subtle pulsing wave animation. Clicking it again halts recording.
* **Transcription Toast:** Display a minimalist, temporary notification toast at the bottom of the screen showing a preview of the processed text (e.g., "Voice command detected: 'Add a conversation node'").

---

### 2. High-Accuracy Voice Processing Pipeline
To eliminate parsing errors, the backend pipeline must follow a strict three-step framework:

* **Active Context Injection:** On every voice trigger, the frontend must package the user's spoken text alongside a highly compressed snapshot of the **current canvas state** (a JSON array listing all existing node IDs, current custom titles, node types, and their current $x, y$ coordinate positions on the screen). This ensures the processor can resolve contextual phrases like *"delete the last node I made"* or *"change the text in the welcome box"*.
* **Mandatory Chain-of-Thought Reasoning:** Instruct the parsing LLM to output a hidden `thought` string property *before* it generates the executable actions array. It must state its architectural goal, identify targeted nodes, and verify component existence.
* **Few-Shot Learning Examples:** Hardcode behavioral templates into the system prompt to train the parser on messy, natural human commands:
    * *Input:* "Delete that transfer node we just added, it's not needed."
      * *Output JSON:* `{ "thought": "User wants to remove a recently created transfer node.", "commands": [{ "action": "DELETE_NODE", "node_id": "transfer_node_04" }] }`
    * *Input:* "Link the booking button to that check availability thing we just made."
      * *Output JSON:* `{ "thought": "User wants to bridge the 'Booking' transition path to the 'Check Availability' node.", "commands": [{ "action": "CONNECT_NODES", "from_node_id": "welcome_01", "via_transition": "Booking", "to_node_id": "check_avail_02" }] }`

---

### 3. Comprehensive Command Dictionary & CRUD System Registry
The parser must seamlessly translate natural human intents into these precise frontend tools and canvas modifications:

* **Node Creation Commands:** Map spoken keywords to our exact component types:
    * *Conversation Node* ("say", "speak", "greet", "talk", "message") -> Properties: `title`, `text`
    * *Function Node* ("run function", "check availability", "trigger API") -> Properties: `title`, `function_name`
    * *Call Transfer Node* ("transfer call", "forward to number", "dial out") -> Properties: `title`, `phone_number`
    * *Press Digit Node* ("press key", "dtmf option", "ivr menu") -> Properties: `title`, `digit_options`
    * *Logic Split Node* ("conditional branch", "if else", "route intent") -> Properties: `title`, `conditions`
    * *Agent Transfer Node* ("send to human", "live operator handoff", "escalate") -> Properties: `title`
    * *In-Call SMS Node* ("send text message", "sms during call") -> Properties: `title`, `sms_body`
    * *Extract Variable Node* ("save response", "capture data fields", "remember what they said") -> Properties: `title`, `variable_name`
    * *Code Node* ("custom script block", "run javascript") -> Properties: `title`, `code_snippet`
    * *Ending Node* ("hang up", "terminate call", "stop workflow") -> Properties: `title`
    * *Note Node* ("add a sticky note", "canvas comment", "remind me") -> Properties: `content`
* **Node & Wire Deletion Commands:** * *Delete Node* ("delete node", "remove block", "destroy card", "get rid of the welcome node") -> Action: `DELETE_NODE`
    * *Delete Connection* ("disconnect", "remove wire", "unlink", "break the connection", "cut the line") -> Action: `DELETE_CONNECTION`
* **Property Modifications:** Interpret phrases that update live elements (e.g., *"Change the phone number in the transfer node to +123456789"* or *"Rename that conversation block to Intro Greeting"*).
