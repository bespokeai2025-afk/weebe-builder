BUILD WEBFORM LEAD CAPTURE SYSTEM

IMPORTANT

Do not break existing Leads, Admin, CRM, Smart Dash, GrowthMind, HiveMind, Builder, WhatsApp, HexMail, Retell or provider integrations.

Extend the current lead capture system safely.

OBJECTIVE

Create a webform lead capture system so leads from website forms can land directly inside WEBEE.

There are two use cases:

1. WEBEE's own website "Talk to us" form should create leads inside the WEBEE internal/admin workspace.

2. Customer websites should be able to connect their own forms and send leads into their own WEBEE Leads section.

====================================================

PART 1 — WEBEE INTERNAL TALK TO US FORM

When someone clicks:

Talk to us

on WEBEE website/app marketing pages:

Show a webform.

Fields:

Name
Email
Phone
Company Name
Website
Interested In
Message
Preferred Contact Method
Source Page
UTM Source
UTM Campaign
UTM Medium

On submit:

Create a lead in the WEBEE internal/admin workspace.

Assign source_type:

webee_website_form

Assign source_detail:

talk_to_us

Assign lead_status:

new

Assign owner/email notification to:

[admin@webespokeai.com](mailto:admin@webespokeai.com)

Do not send this lead into the submitting user's workspace unless they are already a logged-in customer submitting from their own workspace.

====================================================

PART 2 — CUSTOMER WEBFORM CONNECTOR

Create a section inside WEBEE:

/settings/webforms

or:

/leads/webforms

Purpose:

Allow users to connect their own website forms to WEBEE.

Users should be able to create:

Webform Endpoint

Each endpoint gets a unique URL like:

https://webeebuilder.com/api/public/webforms/{form_token}

The customer can paste this URL into:

Website form
WordPress
Webflow
Wix
Framer
GoHighLevel
Zapier
Make
Custom website

====================================================

PUBLIC ENDPOINT

Create:

POST /api/public/webforms/:formToken

Accept:

JSON

application/x-www-form-urlencoded

multipart/form-data if simple fields only

Validate:

form token exists

status = active

allowed domain if configured

====================================================

LEAD CREATION LOGIC

On valid submission:

1. Map fields.
2. Create or update lead by email/phone duplicate check.
3. Store raw payload.
4. Store mapped payload.
5. Add entity note:
   "Lead created from webform: {form name}"
6. Trigger notification if configured.
7. Optionally trigger GrowthMind/HiveMind event.

====================================================

SECURITY

Do not expose workspace IDs in public form URLs.

Use unguessable form_token.

Rate-limit submissions.

Spam protection:

honeypot field

optional turnstile/captcha later

allowed domains

Do not store passwords or sensitive payment fields.

====================================================

SUCCESS CRITERIA

WEBEE Talk to Us form creates leads in WEBEE internal/admin workspace.

Customer webforms create leads in the correct customer workspace.

Every lead has source_type and source_detail.

UTM data is captured.

Users can create and manage webform endpoints.

HiveMind and GrowthMind understand webform leads.

Admin receives notifications for WEBEE website enquiries.

Generate implementation plan first.

Then implement safely.
