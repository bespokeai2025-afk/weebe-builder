Please further develop the SystemMind Agent Workflow Builder so it works more like an intelligent Replit-style workspace for building, testing, re-prompting, fixing, approving, and deploying agent workflows.

The goal is:

When a user asks SystemMind to build an agent or workflow from the Agent Builder page, SystemMind should generate the setup, but if the agent/workflow is not correct, the user should be able to keep prompting SystemMind inside a dedicated workspace until it works properly.

Once the user is happy, they should click an Apply / Deploy button, and the system should apply the workflow, automate the Go Live process where appropriate, save the workflow to the client’s Workflows page, and allow future edits.

This must be workspace-specific and must never change the global WEBEE codebase or affect other users.

---

## 1. Main feature: SystemMind Build Workspace

Add a dedicated SystemMind Build Workspace for agent and workflow creation.

When a user uses the Agent Builder page and selects or prompts SystemMind to build an agent/workflow, open a new sub-tab in the left-hand Builder platform navigation.

Suggested sub-tab name:

* SystemMind Build
* Build Workspace
* Agent Build Workspace
* Workflow Build Studio

This workspace should stay attached to the current agent/workflow being built.

It should allow the user to:

* see the generated agent setup
* see the generated workflow
* test the generated setup
* prompt SystemMind again to fix or improve it
* view change history
* compare versions
* approve changes
* apply changes
* deploy/go live when ready

---

## 2. Re-prompting and iterative fixing

The user should be able to keep giving SystemMind instructions until the workflow is correct.

Example flow:

1. User says:
   “Build me a WhatsApp qualification agent for estate agency leads.”

2. SystemMind generates:

   * agent prompt
   * workflow steps
   * variables
   * post-call/post-chat extraction
   * follow-up rules
   * WhatsApp setup requirements
   * testing checklist

3. User tests it and says:
   “This is wrong, make the qualification questions shorter and add budget before booking.”

4. SystemMind creates a new version.

5. User says:
   “Now add a follow-up sequence if they do not respond.”

6. SystemMind creates another version.

7. User clicks:
   “Apply”
   or
   “Apply & Go Live”

8. The approved setup is applied to the actual Agent Builder / Workflow system.

9. The final workflow is saved into the client’s Workflows page.

The Build Workspace should support unlimited or configurable iterative prompt runs, subject to workspace usage/billing limits.

## 4. Apply button

Add an Apply button inside the SystemMind Build Workspace.

When clicked, Apply should:

1. Validate the generated setup.
2. Check required fields are present.
3. Check workspace_id scoping.
4. Check credentials required.
5. Check provider setup requirements.
6. Check if approval is needed.
7. Save the approved setup to the correct workspace.
8. Update the Agent Builder configuration.
9. Create or update the workflow in the Workflows page.
10. Create an audit log.

Apply must not immediately activate high-risk workflows unless the user/admin confirms.

High-risk examples:

* outbound calling
* WhatsApp broadcasts
* SMS campaigns
* email campaigns
* CRM sync rules
* billing/usage changes
* provider credential changes
* webhook changes
* live customer messaging

For high-risk changes, show:

“This workflow affects live customer communication and requires approval before going live.”

---

## 5. Apply & Go Live button

Add an optional Apply & Go Live button when safe.

This should appear only when:

* the user has permission
* required credentials are present
* all validation checks pass
* no high-risk approval is missing
* the agent/workflow is ready
* the workflow belongs to the current workspace_id

When clicked, Apply & Go Live should:

1. Apply the approved version.
2. Save the workflow to the client’s Workflows page.
3. Trigger the existing Go Live logic in the Agent Deploy tab.
4. Mark agent/workflow status as active/live.
5. Create an audit log.
6. Show success state.

Important:

Do not duplicate the Go Live logic. Reuse the existing Go Live/deploy service if it exists.

Do not bypass existing deployment checks.

Do not break existing Agent Deploy tab behaviour.
