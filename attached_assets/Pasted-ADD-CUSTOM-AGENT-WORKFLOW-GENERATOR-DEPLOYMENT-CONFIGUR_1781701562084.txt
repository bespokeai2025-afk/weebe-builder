ADD CUSTOM AGENT WORKFLOW GENERATOR + DEPLOYMENT CONFIGURATOR VIA SYSTEMMIND

IMPORTANT

Do not break existing Builder workflows.

Do not modify existing Retell, HyperStream, VoxStream or deployed agent flows.

Do not deploy automatically.

Do not overwrite existing canvases without confirmation.

SystemMind must create drafts, validations and admin change requests only.

Live deployment must continue using the existing Go Live pipeline.

====================================================

OBJECTIVE

Add a Custom agent type in the Builder Global Panel that supports two user scenarios:

1. User needs SystemMind to build a new workflow from a description.

2. User already has an agent script/workflow and only needs deployment configuration, post-call data extraction, CRM mapping, webhook mapping, and Go Live readiness.

====================================================

CUSTOM AGENT TYPE FLOW

In Builder Global Panel:

Agent Type:
Custom

Show two options:

OPTION A:
Build Workflow From Description

OPTION B:
Configure Deployment From Existing Script

====================================================

OPTION A — BUILD WORKFLOW FROM DESCRIPTION

User enters:

Workflow description

Example:

"I want a rebooking workflow. If a caller asks for a callback, collect preferred date and time, check availability, confirm the callback, update CRM, and send confirmation."

SystemMind generates:

nodes_json
edges_json
variables_json
tools_json
required_integrations_json
missing_capabilities_json
validation_results_json

Then imports as Builder draft.

====================================================

OPTION B — CONFIGURE DEPLOYMENT FROM EXISTING SCRIPT

User may already have:

A script
A PDF
A conversation flow
A Retell-style prompt
A call script
A qualification questionnaire
A webhook spec
A list of post-call fields

SystemMind should not rebuild the conversation unless needed.

Instead, SystemMind should analyse the existing script and create:

Deployment configuration
Post-call extraction schema
Variable mapping
CRM mapping
Webhook mapping
Calendar mapping
Call outcome mapping
Sentiment/outcome rules
Go Live validation checklist

====================================================

BUILDER INTEGRATION

For Option A:

Import generated nodes/edges into canvas.

For Option B:

Create deployment configuration panel.

Attach extraction schema and mappings to the agent.

Do not force a visual node flow if the user already has a prompt/script-based agent.

====================================================

GO LIVE BEHAVIOUR

When user clicks Go Live:

Run existing deployment path.

If workflow is node-based:

Deploy normal Builder flow.

If script/prompt-based:

Deploy using prompt/script configuration plus extraction schema.

If validation fails:

Do not deploy.

Create SystemMind log.

Create admin change request if missing platform capability is required.

====================================================

SAFETY RULES

SystemMind may:

Create workflow drafts

Create deployment configs

Create extraction schemas

Create CRM mappings

Create webhook mappings

Create admin change requests

SystemMind may not:

Deploy automatically

Modify live agents silently

Change phone routing directly

Change Retell config directly outside Go Live

Start outbound calls

without explicit user approval and existing Go Live validation.

====================================================

SUCCESS CRITERIA

Custom agent type supports both workflow creation and deployment configuration.

Users with existing scripts do not need to rebuild the whole flow.

Post-call data extraction is generated automatically.

CRM/webhook mappings are generated automatically.

Go Live remains controlled and safe.

Existing Retell and other providers are unaffected.

Failures create logs and billable admin change requests.

Generate implementation plan first.

Audit current Builder Global Panel, script import, post-call extraction, Retell deploy and Go Live validation before coding.
