Please audit the current SystemMind, Workflow Builder, Agent Builder, People section, CRM, and campaign filter setup, then extend SystemMind so it can safely create workspace-specific People subpages and campaign filters.

The goal is to make SystemMind powerful, similar to Replit, but safely constrained to the current user’s workspace.

SystemMind should be able to inspect the current setup, understand what exists, suggest improvements, create new workspace-level CRM views/subpages/filters, and configure campaign filters — but it must never break global app pages, other workspaces, WBAH, existing agents, existing workflows, or existing CRM logic.

---

## 2. Main feature: workspace-specific People subpages

Allow SystemMind to create new subpages or saved views inside the People section for the current workspace only.

These should be workspace-configured views, not new global hardcoded app pages unless absolutely required.

Examples of People subpages:

* Hot Leads
* Booked Appointments
* Positive Leads
* Neutral Leads
* Negative Leads
* Callback Requests
* No Answer
* Needs Follow Up
* Qualified Sellers
* Qualified Buyers
* Webform Leads
* CRM Imported Leads
* WhatsApp Leads
* Campaign Leads
* Rebookings
* Do Not Contact
* High Value Leads
* Custom Client View

The user should be able to ask SystemMind:

* “Create a People subpage for booked appointments.”
* “Create a view for positive sentiment leads.”
* “Create a page for callback requests.”
* “Create a campaign filter for webform leads only.”
* “Create a subpage for leads where budget is over £500,000.”
* “Create a page for negative sentiment and show negative reason.”
* “Make a page for leads that need WhatsApp follow-up.”

SystemMind should turn this into a workspace-specific saved view.

---

## 3. Do not create unsafe global pages by default

Do not modify the global route structure for every user unless admin explicitly chooses to create a global feature.

Default behaviour should be:

* create workspace-specific saved view
* show it as a subpage/tab in the People section for that workspace only
* store configuration in the database
* do not touch other workspaces
* do not alter global navigation for everyone

If a true app-level route is needed, SystemMind should create a proposed developer task, not silently edit production routing.

## 6. Campaign filter builder

Allow SystemMind to create campaign filters for the current workspace.

Example filters:

* call only new webform leads
* call only positive/neutral leads
* exclude booked appointments
* exclude do-not-contact
* exclude already called today
* exclude active follow-up sequence
* call only leads with phone number
* call only leads created in last 7 days
* call only CRM imported leads
* call only leads from selected source
* call only leads with status `ready_to_call`
* call only leads not in another active campaign
* call only high-priority leads
* call only callback requests due today

Campaign filters should be usable by:

* Agent Builder
* Workflows page
* Campaigns page
* Follow Up Centre
* SystemMind Build Workspace
* Agent Deploy tab

---

## 7. Duplicate prevention and safety

Campaign filters must include duplicate prevention options.

SystemMind should ask:

* Should booked leads be excluded?
* Should opt-out leads be excluded?
* Should negative leads be excluded?
* Should leads already in another campaign be excluded?
* Should leads called today be excluded?
* Should leads with active follow-up be excluded?
* Should leads without phone numbers be excluded?
* Should leads outside calling hours be excluded?
* Should max calls per day apply?

Default safe settings:

* exclude booked leads
* exclude do-not-contact
* exclude opted-out leads
* exclude active campaign leads
* exclude leads without phone number
* respect workspace call caps
* respect campaign call caps

## 8. SystemMind prompt behaviour

SystemMind should be able to receive a user prompt such as:

“Create a People page for booked appointments and make a campaign filter that only calls positive webform leads who have not booked yet.”

SystemMind should respond with a draft plan:

* People view name
* People view filters
* columns to show
* campaign filter name
* campaign filter rules
* excluded lead types
* affected workspace
* risk level
* expected result
* approval required

Then allow:

* Save as draft
* Apply
* Test filter
* Edit rules
* Cancel

Do not silently apply if the filter affects active campaigns.

---

## 9. Filter dry-run

Before applying a People view or campaign filter, run a dry-run.

Dry-run should show:

* number of matching people/leads
* example matching records
* number excluded
* reasons excluded
* whether any active campaign is affected
* whether any booked/opted-out/negative leads would be included
* warnings
* estimated call volume if used in campaign

No calls or updates should happen during dry-run.

## 12. Permissions

Only authorised users should create or activate views/filters.

Suggested permissions:

* Workspace Admin: create, apply, activate, archive
* Manager: create draft, request approval
* Agent Builder user: create draft only
* Viewer: view only

High-risk filters that affect active campaigns should require admin approval.

## 13. SystemMind should be powerful but not dangerous

SystemMind should be able to:

* inspect current workspace data structure
* suggest new People views
* create workspace-specific CRM views
* create campaign filters
* create custom fields if approved
* test filters with dry-run
* link views to workflows
* link views to campaigns
* save versions
* rollback changes
* explain what it created
* improve views from user prompts

SystemMind must not:

* edit the global application route structure from a user prompt
* affect other workspaces
* change WBAH unless logged into WBAH
* change standard accounts when working in WBAH
* delete fields globally
* rename fields globally
* alter active campaigns without approval
* include opted-out leads by default
* include booked leads by default
* include do-not-contact leads by default
* bypass Supabase RLS
* bypass workspace_id checks

---

## 14. Versioning and rollback

Every People view and campaign filter should have versions.

When edited:

* create new version
* keep old version
* allow rollback
* audit who changed it
* audit SystemMind prompt used
* audit before/after filter config

Do not destroy old config.
