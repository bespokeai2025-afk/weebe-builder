Step 6: API authentication

Use the existing Webuyanyhouse API authentication flow:

POST /admin/auth/request-otp
POST /admin/auth/verify-otp
POST /admin/refresh-token

Store accessToken and refreshToken server-side only against the Webuyanyhouse workspace.

Never expose tokens in frontend code.
Never store tokens in localStorage.
Never log tokens.

All protected API calls must send:

Authorization: Bearer <accessToken>

If any request returns 401:
- call refresh-token
- save the new accessToken
- retry once
- if it fails again, mark the integration disconnected
