2. Authentication flow

Add a secure authentication flow for WebespokeAI Enterprise:

Step 1:
Admin enters email.

Step 2:
Call:
POST /admin/auth/request-otp

Step 3:
Show OTP input.

Step 4:
Call:
POST /admin/auth/verify-otp

Step 5:
Save returned:
accessToken
refreshToken

Tokens must be stored server-side only.
Do not expose tokens to the browser.
Do not store these tokens in localStorage.

If the app already has a workspace_settings or integration_settings table, store the encrypted tokens there against:
integration_key = "webespoke_enterprise"
client_name = "Webuyanyhouse"

If no suitable table exists, create one using Supabase migration:

enterprise_integrations:
- id uuid primary key
- workspace_id uuid nullable
- client_name text not null
- integration_key text not null
- access_token text
- refresh_token text
- user_payload jsonb
- status text default 'disconnected'
- created_at timestamptz default now()
- updated_at timestamptz default now()

Add RLS so only the correct workspace/admin can access it.

3. Auto refresh token logic

Every authenticated API request must include:

Authorization: Bearer <accessToken>

If any request returns 401:
- call POST /admin/refresh-token with the stored refreshToken
- save the new accessToken
- retry the original request once
- if retry still fails, mark integration as disconnected and show a reconnect message
