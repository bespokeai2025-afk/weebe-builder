SQL MIGRATION & MANUAL SQL AUDIT

IMPORTANT

Do NOT execute any SQL.

Do NOT apply any migrations.

Do NOT modify the database.

Do NOT create, alter or drop any tables.

Do NOT change production data.

This is a READ-ONLY audit.

---

GOAL

Determine whether there are any SQL scripts, manual SQL statements, Supabase SQL Editor commands, seed scripts or database changes that still require manual execution outside of the normal migration process.

The objective is to identify anything that cannot be completed automatically by the migration system.

---

CHECK THE ENTIRE PROJECT

Search the entire codebase for:

* Raw SQL files
* .sql files
* SQL Editor scripts
* Manual SQL instructions
* README database setup instructions
* Deployment documentation
* Database setup guides
* Seed scripts
* Backfill scripts
* One-time migration scripts
* Manual repair scripts
* SQL embedded inside TypeScript
* SQL embedded inside server functions
* SQL embedded inside documentation
* SQL generated by previous development work

---

CHECK MIGRATIONS

Review every migration.

Identify if any migration requires manual intervention.

Examples:

* must be run in Supabase SQL Editor
* requires service_role
* requires pg_cron
* requires extension enablement
* requires manual data migration
* requires backfill
* requires environment variables first
* requires external configuration
* requires production-only execution

---

CHECK SUPABASE

Verify whether anything still requires manual SQL inside Supabase.

Examples:

* Extensions
* RLS policies
* Functions
* Triggers
* Views
* Materialized views
* Cron jobs
* Scheduled jobs
* Index rebuilds
* Constraints
* Foreign keys
* Storage buckets
* Storage policies
* Auth triggers
* Realtime configuration
* pgvector configuration

CHECK MANUAL CONFIGURATION

Identify anything that still requires manual setup such as:

* SQL Editor commands
* Service Role configuration
* RLS policy creation
* Trigger installation
* Function installation
* Cron activation
* Scheduled jobs
* Views
* Materialized views
* Seed data
* Default records
* Platform settings
* Workspace defaults

---

VERIFY

Confirm whether:

All required SQL has already been executed.

OR

Specific SQL still needs to be run manually.

---

RETURN

Produce a report with the following sections:

1. Manual SQL Required
2. SQL Already Applied
3. SQL That Can Be Ignored
4. SQL That Has Been Superseded
5. SQL That Should Never Be Applied
6. SQL That Requires Manual Review
7. SQL That Must Be Executed In Supabase SQL Editor
8. SQL Required Before Production
9. SQL Required After Deployment
10. SQL Required For Future Features

For every item include:

* File name
* SQL object affected
* Reason
* Risk
* Whether it is mandatory or optional
* Whether it is safe to execute
* Dependencies
* Exact execution order if required

Finally answer clearly:

1. Is there ANY SQL that still requires manual execution?

2. If YES, list ONLY the SQL that genuinely still needs to be run.

3. If NO, explicitly state:

"THE DATABASE IS FULLY RECONCILED. NO MANUAL SQL EXECUTION IS REQUIRED."
