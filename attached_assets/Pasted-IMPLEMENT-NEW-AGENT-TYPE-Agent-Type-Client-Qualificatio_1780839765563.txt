IMPLEMENT NEW AGENT TYPE

Agent Type:
Client Qualification

This is a separate workflow from Lead Generation.

PURPOSE

The Client Qualification agent is responsible for contacting existing prospects and determining whether they qualify based on predefined business criteria.

The agent should be able to call contacts from either:

1. Data Section (CSV Sources)
2. Leads Section (Existing Leads)

The source must be selectable during agent configuration.

SOURCE SELECTION

Within Agent Builder:

Agent Type → Client Qualification

Add:

Lead Source

Options:

○ Data Section
○ Leads Section

User must select one source before deployment.

WORKFLOW CONFIGURATION

If Source = Data Section

Display:

* Dataset Selection
* Variable Mapping
* Calling Rules
* Qualification Rules

Use the existing Data Section architecture.

If Source = Leads Section

Display:

* Lead List Selection
* Lead Filters
* Calling Rules
* Qualification Rules

The agent should call through existing leads.

LEADS SECTION INTEGRATION

Add a new action within Leads.

Example:

Select Leads
↓
Actions
↓
Assign Qualification Agent

User selects:

* Qualification Agent
* Calling Schedule
* Daily Limits
* Retry Rules
* Qualification Criteria

This creates a Qualification Campaign.

QUALIFICATION ENGINE

Allow users to define qualification criteria.

Examples:

Budget Confirmed
Decision Maker
Business Size
Location
Urgency
Current Provider
Interest Level

Builder should support:

Required Qualification Conditions

Example:

Budget = Yes
Decision Maker = Yes
Interest Level = Medium+

Result:

Qualified = True

QUALIFICATION OUTCOMES

Possible outcomes:

Qualified
Partially Qualified
Not Qualified
Call Back Required
Voicemail
Wrong Number
Do Not Contact

Store outcome against contact.

POST CALL ANALYSIS

After every call:

Retrieve:

* Transcript
* Recording Metadata
* Call Outcome

Run Qualification Analysis.

Generate:

{
"qualification_status":"qualified",
"sentiment":"positive",
"budget_confirmed":true,
"decision_maker":true,
"urgency":"high",
"interest_level":"high",
"next_step":"sales_followup",
"summary":"Prospect meets qualification criteria and requested pricing information."
}

SHARED INFRASTRUCTURE

Reuse existing:

* Variable Mapping Engine

Do not duplicate existing systems.

ARCHITECTURAL GOAL

Create a Client Qualification workflow capable of:

Data Section → Qualification Calls → Calls Section → Qualified Section

OR

Leads Section → Qualification Calls → Calls Section → Qualified Section

using a single shared qualification engine while maintaining full multi-tenant isolation.
