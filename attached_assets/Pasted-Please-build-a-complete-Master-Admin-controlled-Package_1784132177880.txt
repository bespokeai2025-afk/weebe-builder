Please build a complete Master Admin controlled Package Access, Staff Access, Notification, White Label, and Reseller/Client Account system for WEBEE.

This must combine:

1. Package-based access for all new signups.
2. Staff logins as chargeable add-ons.
3. Panel/page/action visibility controls.
4. Account Settings notification preferences.
5. White Label package controls.
6. Reseller ability for selected clients to create white-label client accounts.
7. Master Admin oversight over all workspaces, reseller accounts, child client accounts, usage, billing, branding, notifications, and access.
8. Standard WEBEE automated emails sent through the admin WEBEE Resend account by default.
9. Optional workspace/reseller custom email provider settings if they prefer to send branded emails from their own domain/provider.

Everything must be workspace-scoped, tenant-safe, auditable, reversible, and enforced on the backend.

---

# 1. Core access model

Create a clear hierarchy:

## WEBEE Master Admin

The top-level owner/admin layer.

Can:

* manage all packages
* manage all workspaces
* manage direct clients
* manage white label clients
* manage reseller accounts
* manage reseller child workspaces
* view usage across all accounts
* control package entitlements
* grant/revoke white label access
* grant/revoke reseller access
* view billing/usage
* view audit logs
* suspend workspaces
* override access
* configure default notification/email settings
* configure platform-wide Resend/admin email account

## Direct WEBEE Client Workspace

A normal client account sold directly by WEBEE.

Access is controlled by:

* package
* add-ons
* staff seats
* user role
* page/action permissions
* panel visibility

## White Label / Reseller Workspace

A client who has permission to create their own branded client accounts.

Access is controlled by:

* package
* white label entitlement
* reseller entitlement
* add-ons
* limits
* Master Admin approval/override

## Reseller Child Client Workspace

A workspace created by a reseller for their own client.

This workspace should:

* inherit reseller branding where configured
* be linked to the reseller parent workspace
* still be visible to WEBEE Master Admin
* still be auditable by WEBEE Master Admin
* still have package/add-on/usage tracking
* still respect WEBEE platform security
* never bypass Master Admin control

# 11. Email sending priority

When sending automated emails, choose provider in this priority:

1. Child workspace custom provider, if active and package allows.
2. Reseller parent custom provider, if child inherits parent email mode and package allows.
3. WEBEE admin Resend account as default fallback.

Never fail campaign execution because an email failed.

If custom provider fails:

* log error
* mark notification failed
* optionally fallback to WEBEE default if workspace setting allows fallback
* notify admin in-app
* do not expose secrets

# 21. SystemMind behaviour

SystemMind must respect package, role, and reseller permissions.

Examples:

* If user asks to create child client account but reseller feature is not enabled, SystemMind should show upgrade/admin approval prompt.
* If user lacks White Label permission, SystemMind should refuse.
* If user lacks package access for custom email provider, SystemMind should say it is locked.
* If user can draft but not apply, SystemMind should create a draft and request approval.
* If reseller child account limit is reached, SystemMind should block and suggest add-on/upgrade.
* If custom email provider fails, SystemMind should suggest fix without exposing secrets.

SystemMind must never change global WEBEE settings from a workspace prompt.

# 24. WBAH rule

WBAH must remain isolated.

Do not change WBAH data access unexpectedly.

Do not copy WBAH branding, filters, roles, staff settings, reports, notifications, or provider settings to other workspaces.

If WBAH has special access/admin requirements, preserve them and layer the new permission system carefully.
