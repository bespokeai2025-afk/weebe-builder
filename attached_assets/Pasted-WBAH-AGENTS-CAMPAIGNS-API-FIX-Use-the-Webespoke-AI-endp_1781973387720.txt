WBAH AGENTS + CAMPAIGNS API FIX

Use the Webespoke AI endpoints below for agent and campaign syncing.

AGENTS

Use:

GET /agent/get-list

Purpose:
List active agents.

Use optional query:

?campaign_only=true

when only agents available for campaigns are needed.

Also audit:

GET /agent/get-agents-with-voicemail

Purpose:
List agents with voicemail configuration.

POST /agent/update

Purpose:
Save selected/active agents.

PATCH /agent/rename/:id

Purpose:
Rename agent display name.

PATCH /agent/voicemailSetting/:id

Purpose:
Enable/edit voicemail message.

CAMPAIGNS

Use:

GET /campaigns

Purpose:
List campaigns with status.

POST /campaigns

Purpose:
Create campaign.

GET /campaigns/:id

Purpose:
Get one campaign by ID.

PATCH /campaigns/:id

Purpose:
Update campaign configuration.

PATCH /campaigns/:id/pause

Purpose:
Pause running campaign.

PATCH /campaigns/:id/resume

Purpose:
Resume paused campaign.

PATCH /campaigns/:id/voicemail

Purpose:
Toggle voicemail behaviour.

DELETE /campaigns/:id

Purpose:
Soft-delete campaign while keeping history.

====================================================

EXPLAIN THE DIFFERENCE

AGENT:
The AI caller / voice agent configuration.

Examples:
Receptionist agent
Lead generation agent
Qualification agent
Callback agent
Voicemail-enabled agent

CAMPAIGN:
The calling workflow that uses an agent to contact leads.

Examples:
Call these CRM leads
Call only positive/neutral leads
Call only pending callbacks
Pause campaign
Resume campaign
Change voicemail setting
Change schedule

Agent = who is speaking.

Campaign = who they are calling, when, and under what rules.

====================================================

SYNC RULES

When syncing agents:

Use /agent/get-list for normal agent list.

Use /agent/get-list?campaign_only=true for campaign selection dropdowns.

Use /agent/get-agents-with-voicemail where voicemail settings are displayed or edited.

When syncing campaigns:

Use GET /campaigns for campaign list.

Use GET /campaigns/:id for detailed campaign view if needed.

When updating campaign:

Use PATCH /campaigns/:id.

When pausing:

Use PATCH /campaigns/:id/pause.

When resuming:

Use PATCH /campaigns/:id/resume.

When toggling voicemail:

Use PATCH /campaigns/:id/voicemail.

====================================================

DATA CONSISTENCY

Do not mix agents and campaigns.

Do not treat an agent as a campaign.

Do not treat a campaign as an agent.

Campaigns should reference agents by external_agent_id.

Agent dropdowns in campaign creation should use active/campaign-capable agents only.
