Please extend the current People Views, Campaign Filters, SystemMind, and Campaigns implementation.

I want SystemMind to be able to safely create and update filters for each relevant page, not only People custom views. I also want automatic campaign reporting as standard every time a campaign is activated, every time a campaign fails, and every time campaign KPI outcomes are available.

This must be fully workspace-scoped and must never affect another user’s workspace.

---

# 1. Page-level filter configuration

Allow SystemMind to create, update, dry-run, version, and roll back filters for each major page.

Supported pages:

* People
* Leads
* Qualified
* Calls
* Data
* Campaigns
* Follow Up Centre
* Workflows
* Agent performance / Analytics where relevant
* Custom People Views
* Custom campaign views

These filters should be database-backed workspace configs, not hardcoded global page changes.

SystemMind should be able to respond to prompts like:

* “Add a filter to Leads for positive sentiment.”
* “Create a Qualified page filter for booked appointments.”
* “Add a Calls filter for voicemail only.”
* “Add a Data page filter for webform leads.”
* “Create a Campaigns filter for failed campaigns.”
* “Show only leads from this campaign.”
* “Create a filter for leads that need WhatsApp follow-up.”
* “Add negative reason as a filter on Calls and Leads.”

# 18. Safety

Do not break existing campaign execution.

Campaign reports must be additive.

If report generation fails, campaign execution must not fail because of reporting.

Wrap report generation safely and log report errors separately.

Never allow a report job to:

* trigger calls
* change campaign status
* update CRM records
* send messages
* alter filters
* alter agents
* alter workflows

Reports are read-only summaries unless the user explicitly asks SystemMind to create a fix draft.

---

# 19. Workspace isolation

Every filter, report, KPI, recommendation, and notification must be scoped to current workspace_id.

SystemMind must never:

* read another workspace’s campaign reports
* create filters for another workspace
* apply WBAH reports to standard workspaces
* apply standard account reports to WBAH
* change global filters from a workspace prompt
* create global report views from a workspace prompt

Reject if workspace_id is missing or mismatched.
