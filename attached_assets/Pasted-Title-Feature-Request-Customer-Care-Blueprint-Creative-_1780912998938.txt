Title: Feature Request - Customer Care Blueprint & Creative Extrapolation Matrix

Please expand the "Creative Extrapolation Engine" inside the Macro Mode (Webee Build Mode) backend processor to natively support and creatively unpack Customer Care, Support, and Helpdesk workflows. When a user requests a "customer care" or "support" flow with minimal business context, the engine must autonomously brainstorm realistic support categories, ticket triage paths, and high-fidelity troubleshooting dialogues specific to that industry sector.

### 1. The Master Customer Care / Support Playbook Blueprint
Hardcode this mandatory structural baseline into the system prompt context for the macro flow generator. When a user requests a customer care/support system, the AI must use this framework as its architectural foundation:

* **Stage 1: Greeting & Authentication (Conversation Node -> Extract Variable Node)**
    * *Rule:* Welcome the customer warmly, introduce the AI support assistant, and immediately capture an authentication token (e.g., Order ID, Account Number, or Email address) before diagnosing the issue.
* **Stage 2: Issue Triage Matrix (Logic Split Node)**
    * *Rule:* Intuitively branch the conversation into 3–5 core, domain-specific support channels (e.g., Technical Troubleshooting, Billing/Invoices, Returns/Refunds, or General FAQs).
* **Stage 3: Automated Resolution & KB Lookup (Function Node -> Logic Split Node)**
    * *Rule:* For standard issues (like tracking an order or resetting a password), queue a Function node to simulate a live database query or Knowledge Base API lookup. Route based on the output ("Article Found" vs "Escalation Required").
* **Stage 4: Deflection & Fallback Messaging (In-Call SMS Node)**
    * *Rule:* If the user needs an immediate step-by-step tutorial or a secure password reset link, utilize an In-Call SMS node to push the URL text instantly to their phone to keep the call efficient.
* **Stage 5: Graceful Escalation (Agent Transfer Node / Call Transfer Node)**
    * *Rule:* If the issue is marked as critical, unresolved by the KB query, or billing-sensitive, map it cleanly to a live operator handoff or queue routing node.
