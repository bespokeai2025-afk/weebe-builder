Please upgrade the WEBEE Analytics Centre into a much deeper business intelligence and reporting centre.

The Analytics Centre should analyse calls, leads, qualified leads, campaigns, agents, workflows, follow-ups, bookings, costs, revenue, ROI, and AI department recommendations.

It should also generate reports and send automated campaign result reports using the WEBEE admin Resend account by default, or a workspace/reseller custom email provider if configured and allowed by package.

Everything must be workspace-scoped, package-gated, role-permission checked, and safe.

---

It should include:

# 15. Automated campaign reports

Campaign reports should be generated automatically.

Generate reports when:

* campaign is activated
* campaign is paused
* campaign is completed
* campaign fails
* campaign is cancelled
* campaign is retried
* campaign has no eligible leads
* campaign hits safety cap
* campaign hits daily call cap
* campaign produces KPI results
* provider/API error occurs
* workflow error occurs

Campaign report generation must be additive.

If report generation fails, campaign execution must not fail.

Log report generation failures separately.

---

# 16. Automated report sending

Reports should be sendable automatically based on notification settings.

Use provider priority:

1. Workspace custom email provider if active and package allows.
2. Reseller parent custom email provider if child inherits parent email mode and package allows.
3. WEBEE admin Resend account as default fallback.

Default should be WEBEE admin Resend.

Use:

* `WEBEE_RESEND_API_KEY`
* `WEBEE_DEFAULT_FROM_EMAIL`
* `WEBEE_DEFAULT_FROM_NAME`

Do not expose the WEBEE Resend key.

Notification/report settings should decide:

* which reports are emailed
* who receives them
* frequency:

  * immediate
  * hourly digest
  * daily digest
  * weekly digest

If email fails:

* mark as failed
* log error
* show in-app warning
* do not break campaign execution

# 23. Safety

Analytics must be read-only unless the user explicitly creates a draft action from an insight.

Analytics should never directly:

* trigger calls
* send messages
* alter campaigns
* alter agents
* alter workflows
* change CRM records
* change package/access settings

Any action from analytics should create a draft through SystemMind and require approval.

---

# 24. Workspace isolation

All analytics must be scoped to workspace_id.

Do not cross workspaces except Master Admin views.

Do not expose WBAH analytics to standard workspaces.

Do not expose reseller child analytics to sibling child workspaces.

Reseller parent can only see child workspace analytics if permission and package allow.

Master Admin can see all.
