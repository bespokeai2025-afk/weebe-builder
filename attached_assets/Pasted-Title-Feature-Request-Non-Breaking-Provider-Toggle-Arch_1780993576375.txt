This implementation must treat our existing Retell AI setups as a locked, read-only gold standard—guaranteeing that all live agents AND all newly created future agents remain completely native to Retell unless explicitly opted out.

### 1. Dedicated "Switch Providers" Section & Hidden Admin Credentials
* **The UI Section:** Create a distinct, prominent section in the Agent Settings sidebar or Advanced Configuration tab titled: "Voice Infrastructure Routing". 
* **The Switch:** Add a clear toggle or segmented button group: `[ Retell AI Engine (Standard Production) | Native OpenAI Fast Engine (Enterprise Experimental) ]`. 
* **Strict Defaults for Future Scalability:** Ensure that when ANY new agent is created now or in the future, the database column 'voice_provider' automatically defaults to 'RETELL'.

Instead, display a system status badge: "Routed via Master Admin Enterprise Line". Behind the scenes, the backend server must automatically bind the platform's root environmental variable ('process.env.MASTER_OPENAI_API_KEY') to this agent's backend calls. The client never sees this key.

* This ensures that call logs generated by the custom provider update our dashboard sections seamlessly, matching the visual layouts of your current setup perfectly.
