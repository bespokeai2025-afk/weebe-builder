# FUNCTIONAL REGRESSION CHECK

Do not assume that because something exists in the design canvas it exists in the application.

Verify using actual rendered components.

For every route:

1. Open the page.
2. List available user actions.
3. Compare against previous implementation.
4. Restore any missing actions.
5. Verify actions render in code.
6. Verify actions are clickable.
7. Verify desktop and mobile behavior.

If an action is missing from the component tree, it must be added.

Do not ship based solely on canvas/mockup appearance.
