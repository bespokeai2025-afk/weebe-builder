Improve the SystemMind Build Console and Agent Builder setup experience so it is no longer messy, vague, or chat-only.

Current problem:
The SystemMind Build Console currently shows vague setup warnings like “required inputs remaining” but it does not clearly show where the user must enter CRM access details, CRM field codes, status trigger fields, variable mappings, webhook details, or calendar details.

The Agent Builder page also has complex scripts/flows with variables, but SystemMind is not scanning the selected agent/script properly and turning those variables into clear mapping requirements.

Goal:
Make SystemMind work like a proper setup console, similar to how Replit asks for missing API keys/secrets with a clear credential input box.

SystemMind should:
1. Detect the selected/linked agent.
2. Scan the agent script, prompt, nodes, tools, functions, post-call extraction, webhook payloads, and templates for variables.
3. Show a clean setup panel with required input boxes.
4. Ask for CRM credentials in a dedicated credential box.
5. Ask for variable mappings in a clear mapping table.
6. Ask for CRM status trigger rules in a clear trigger mapping table.
7. Block Apply / Go Live until every required credential, mapping, and trigger field is complete.
8. Keep everything workspace-scoped and agent-specific.

Do not just add more chat messages. Add proper UI components.

The core user experience should be:

User opens SystemMind Build Console or Agent Builder.
User selects or is already inside an agent.
SystemMind scans the agent.
SystemMind says:

“I found this agent, scanned the workflow, and detected these variables and setup requirements.”

Then SystemMind opens a clear setup panel showing:

- Agent linked
- CRM access required
- Variables detected
- CRM field mappings needed
- Status triggers needed
- Calendar/booking config needed if applicable
- Webhook/API config needed if applicable
- Test payload
- Apply locked until complete
