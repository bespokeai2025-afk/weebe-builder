FIX WEBEE PRODUCTION DEPLOYMENT ARCHITECTURE

IMPORTANT

Do not change product features.

Do not modify HiveMind, GrowthMind, SystemMind, Builder, CRM, Retell, WhatsApp, HexMail, Video Studio, Image Studio or billing logic.

This is infrastructure/deployment hardening only.

OBJECTIVE

Stop building the app directly on the live production EC2 server.

Current risk:

Production deploy runs npm install and npm run build on the live server.
This competes with the running app for CPU/RAM and can cause slowdowns or crashes.

Target:

Build artifact outside production.
Deploy pre-built artifact/container to production.
Allow safe rollback.

AUDIT FIRST

Check current deployment files:

Dockerfile
deploy scripts
GitHub Actions
Nginx config
PM2/systemd config
AWS EC2 setup
DEPLOY_AWS.md
package.json scripts

Identify:

How production deploy currently works
Where npm install runs
Where npm run build runs
How app is restarted
Whether rollback exists
Whether health checks exist

IMPLEMENT SAFELY

Preferred approach:

1. Build Docker image in GitHub Actions.
2. Push image to Amazon ECR.
3. Deploy image to AWS ECS Fargate or App Runner.
4. Keep EC2 only if absolutely required.
5. Add health check before switching traffic.
6. Add rollback to previous image if health check fails.

If ECS/App Runner is too large a change now:

Implement interim safer EC2 deploy:

1. Build artifact in GitHub Actions.
2. Upload build artifact to server.
3. Install production dependencies only.
4. Use release folders:
   /var/www/webee/releases/{timestamp}
   /var/www/webee/current -> active release
5. Switch symlink only after health check passes.
6. Keep previous release for rollback.
7. Restart app only after successful build.
8. Never run npm run build while app is serving production traffic.

ADD

Health check endpoint:

/api/health

Return:

status
version
commit_sha
uptime
database_connection
environment

SUCCESS CRITERIA

Production server no longer runs full app build under live traffic.

Deployments are atomic.

Rollback exists.

Health check protects bad deploys.

No customer-facing downtime during normal deploy.
