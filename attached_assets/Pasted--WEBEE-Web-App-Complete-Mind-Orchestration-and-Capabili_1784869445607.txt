# WEBEE Web App — Complete Mind Orchestration and Capability Audit

You are upgrading the existing WEBEE web application.

Do not rebuild the platform and do not create duplicate versions of existing modules.

The objective is to ensure the WEBEE web application exposes a complete, shared, server-side intelligence and action layer that can also be used by the separate WEBEE mobile application.

The web app must remain the complete management interface, but it must not contain business logic that is inaccessible to mobile.

Follow the attached WEBEE Shared Intelligence and Capability Contract.

## Audit before changing anything

Audit the current implementation of:

* HiveMind

* Notifications
* Task and action system
* Permissions
* Supabase RLS
* Background jobs
* Mobile-facing API routes

Identify:

* Features that exist only inside web components
* Tools implemented only in the web frontend
* Direct database calls that should use services
* Duplicate Mind logic
* Missing workspace scoping
* Missing permissions
* Missing mobile-compatible API endpoints
* Features that display data but cannot perform real actions
* Functions that use mock data
* Functions that report success without backend confirmation

Refactor these into shared services without breaking the existing web UI.

# Cross-Mind orchestration

HiveMind must coordinate the Minds through real tasks and actions.

Example workflows:

## Campaign underperforming

1. HiveMind detects poor campaign conversion.
2. GrowthMind analyses messaging, audience and script.
3. SystemMind checks delivery, API errors and workflow execution.
4. AccountsMind checks cost and return.
5. HiveMind produces one coordinated recommendation.
6. Approved changes are assigned as tasks.
7. Results are measured.

## Client invoice missing

1. AccountsMind identifies that an invoice should exist.
2. HiveMind checks the client register and service status.
3. SystemMind checks whether the invoice-generation workflow failed.
4. HiveMind creates the correct action.
5. An authorised user reviews the invoice.
6. Status is monitored until resolved.

## Qualified lead not followed up

1. HiveMind identifies a qualified lead without activity.
2. It checks the sales pipeline and calendar.
3. GrowthMind supplies relevant messaging.
4. Follow-Up Centre creates the permitted sequence.
5. SystemMind verifies the sequence is operational.
6. HiveMind monitors the outcome.

## Social trend opportunity

1. GrowthMind finds the opportunity.
2. HiveMind checks current product priorities, sales capacity and budget.
3. AccountsMind confirms production and advertising limits.
4. GrowthMind creates a Content Studio project.
5. HiveMind requests approval.
6. Content is published.
7. Performance and revenue are measured.
