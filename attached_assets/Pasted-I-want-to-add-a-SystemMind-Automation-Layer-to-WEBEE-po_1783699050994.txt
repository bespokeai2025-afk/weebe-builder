I want to add a SystemMind Automation Layer to WEBEE, powered by Claude Agent SDK / Claude Code-style agentic tooling, but it must be implemented safely.

Goal:

SystemMind should become intelligent enough to help configure, generate, and automate workspace-specific workflows, agent setups, WhatsApp agent setups, follow-up sequences, and integration blueprints inside the WEBEE platform.

This must not allow SystemMind, HiveMind, or any user interface to directly change the global WEBEE codebase or affect all users.

Critical rule:

SystemMind must only create or update workspace-specific configuration, workflows, prompts, setup tasks, templates, and automation records for the signed-in user’s own `workspace_id`.

It must never make uncontrolled changes across the main WEBEE codebase or across all user workspaces.

---

## 1. Architecture requirement

Please design and implement a safe architecture where SystemMind can use Claude Agent SDK / Claude Code-style automation as a backend intelligence layer.

SystemMind should be able to:

* Read the user’s workspace context.
* Understand current agent setup.
* Generate workflow configs.
* Generate custom prompt workflows.
* Generate WhatsApp agent setup steps.
* Generate WATI / Meta / Twilio setup instructions and config drafts.
* Generate follow-up sequences.
* Generate Follow Up Centre campaigns.
* Convert imported n8n workflows into WEBEE-native workflow blueprints.
* Recommend changes under HiveMind instruction.
* Present changes for user approval before activation.

SystemMind must not directly edit global app source code for normal user actions.

---

## 2. Use Claude safely

Use Claude Agent SDK / Claude Code-style tooling only inside a controlled backend job runner.

Do not run Claude with unrestricted shell access in production.

Do not allow arbitrary user prompts to execute commands.

Do not expose Anthropic API keys or Claude Code credentials to the frontend.

Use server-side environment variables only.

Suggested env variables:

```env
ANTHROPIC_API_KEY=
SYSTEMMIND_CLAUDE_ENABLED=true
SYSTEMMIND_REQUIRE_APPROVAL=true
SYSTEMMIND_SANDBOX_MODE=true
SYSTEMMIND_MAX_ACTIONS_PER_RUN=20
```

If Claude is not configured, SystemMind should show a clear message and continue using the existing SystemMind logic.

---

## 3. Workspace isolation

Every SystemMind-generated object must include:

* `workspace_id`
* `created_by_user_id`
* `source = systemmind`
* `approval_status`
* `version`
* `audit_log_id`

This applies to:

* workflows
* workflow steps
* agent prompts
* WhatsApp agent configs
* WATI setup configs
* Meta WhatsApp setup configs
* Twilio setup configs
* Follow Up Centre sequences
* follow-up campaigns
* imported n8n blueprints
* generated templates
* generated automation rules

No generated object may be created without a workspace_id.

Add backend validation that rejects any SystemMind action where `workspace_id` is missing or does not match the signed-in user.

---

## 4. Approval-first system

SystemMind should not immediately activate generated changes.

Use this lifecycle:

1. `draft`
2. `pending_approval`
3. `approved`
4. `active`
5. `paused`
6. `rejected`
7. `failed`

HiveMind can recommend the action.

SystemMind can build the draft.

The user/admin must approve before activation.

For high-risk actions, require explicit approval.

High-risk actions include:

* sending WhatsApp broadcasts
* activating outbound calling
* changing live agent prompts
* changing provider credentials
* changing webhook URLs
* activating follow-up campaigns
* editing CRM sync rules
* creating automations that message customers
* modifying billing/usage-related settings
